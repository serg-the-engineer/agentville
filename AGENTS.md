# AGENTS.md

Короткая карта репозитория для агентной разработки (agent-legible repo).
Основа подхода: [Harness Engineering](https://openai.com/index/harness-engineering/).

## 1) Что читать первым
1. `PRD.md` — продуктовые правила матча.
2. `ARCHITECTURE.md` — исполнимая архитектура MVP.
3. `docs/EPICS_PLAN.md` — епики и зависимости.
4. `docs/AGENT_DELIVERY_FRAMEWORK.md` — как делить работу между агентами.
5. `docs/QUALITY_AND_DOD.md` — TDD, quality gates, DoR/DoD.
6. `docs/GIT_WORKFLOW.md` — правила веток, PR и merge.
7. `docs/GITHUB_TASKS_PLAYBOOK.md` — практический workflow задач в GitHub.

## 2) Где код
- `src/arena/` — движок, модели, setup, LLM-контракт.
- `tests/` — unit/integration тесты.

## 3) Базовые команды
- Запуск тестов: `uv run --with pytest pytest -q`
- Поиск по коду: `rg "<pattern>" src tests`

## 4) Обязательные правила разработки
- Только TDD: сначала тест (`RED`), затем минимальная реализация (`GREEN`), затем рефакторинг (`REFACTOR`).
- Любая фича начинается с изменения/добавления тестов.
- Любой PR должен содержать тест-план и результаты прогонов.
- Ветка/PR — только одна логическая задача.

## 5) Принципы агентной работы
- Делить задачи на маленькие, контрактно-ограниченные пакеты.
- Держать интерфейсы стабильными; изменения контрактов отдельным PR.
- Писать краткие handoff-заметки: что сделано, что осталось, как воспроизвести.
- Наращивать ограничения через код (тесты, линтеры, типы), а не через длинные текстовые правила.
